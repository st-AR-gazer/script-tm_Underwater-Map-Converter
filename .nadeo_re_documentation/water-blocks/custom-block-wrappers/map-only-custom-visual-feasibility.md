# Map-Only Custom Visual Feasibility

This file is about one practical question:

- how far can custom visuals go while still keeping the result usable on stock clients?

## Current best answer

The wrapper route may help with visuals, but it is not yet a proven complete answer for gameplay water.

The most realistic short-term path is:

- use proven gameplay-water carriers
- hide or simplify their visuals
- keep the map compatible with stock clients

## Why this matters

This is exactly the converter project's sweet spot:

- practical underwater-feeling maps
- without requiring custom environment installs
